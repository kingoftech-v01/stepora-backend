## Critical Findings Requiring Action

### 1. Firebase Service Account Key in Repository (Vuln #115) -- CRITICAL
**File:** `/root/stepora/firebase-service-account.json`
**Issue:** Contains a real Firebase private key for `dreamplaanner-test` project. Even though it is in `.gitignore`, the file exists on disk and may have been committed to git history at some point.
**Action:** Rotate the Firebase service account key immediately. Scan git history with `trufflehog` or `gitleaks`. Use `git-filter-repo` if the key was ever committed.

### 2. Dockerfile Exposes Placeholder Encryption Key (Vuln #124) -- CRITICAL
**File:** `/root/stepora/Dockerfile` line 60
**Issue:** `ARG FIELD_ENCRYPTION_KEY="oWaFzG4bta5qhklgSfvMxFPqwqvy__35IrKHEPURBkg="` bakes a key into Docker image layers. Anyone with access to the ECR image can extract it.
**Action:** Use a dummy/empty placeholder that fails at build time with a clear error, or restructure the build to not require the key during `collectstatic`.

### 3. No Encryption Key Rotation Support (Vuln #125) -- MEDIUM
**Issue:** `encrypted_model_fields` does not support key versioning. If `FIELD_ENCRYPTION_KEY` is compromised, all encrypted data becomes unrecoverable after rotation without a migration script.
**Action:** Document a key rotation procedure. Consider migrating to a library that supports key versioning (e.g., `django-fernet-encrypted-fields` with key rotation).

### 4. No Certificate Pinning on Mobile (Vuln #129) -- MEDIUM
**Issue:** `/root/stepora-mobile` has no SSL pinning. MITM proxies with installed CA certs can intercept all traffic.
**Action:** Implement certificate pinning using a library like `react-native-ssl-pinning` or TrustKit.

### 5. No Constant-Time Token Comparison (Vuln #135) -- MEDIUM
**Issue:** Custom token comparisons (feed tokens, share tokens) may use standard `==` operator, vulnerable to timing attacks.
**Action:** Use `hmac.compare_digest()` for all token/secret comparisons in view code.

### 6. PWA Caches Authenticated API Responses (Vulns #151, #199) -- MEDIUM
**File:** `/root/stepora-frontend/vite.config.js` lines 81-101
**Issue:** Service worker caches all API responses for 24 hours with `StaleWhileRevalidate`. Authenticated user data could leak on shared devices.
**Action:** Exclude authenticated API paths from service worker caching, or add `Cache-Control: no-store, private` headers to authenticated responses from the backend.

### 7. S3 Media Files Publicly Accessible Without Auth (Vuln #177) -- HIGH
**File:** `/root/stepora/config/settings/production.py` line 93
**Issue:** `AWS_QUERYSTRING_AUTH = False` generates direct public S3 URLs for all uploaded media. Anyone with the URL can access files without authentication.
**Action:** Enable `AWS_QUERYSTRING_AUTH = True` (pre-signed URLs with expiration) or implement a proxy endpoint with auth checks.

### 8. No Image Decompression Bomb Protection (Vuln #175) -- MEDIUM
**Issue:** No `PIL.Image.MAX_IMAGE_PIXELS` setting or dimension validation on uploaded images. A crafted image could exhaust server memory.
**Action:** Add `from PIL import Image; Image.MAX_IMAGE_PIXELS = 25_000_000` in settings, and validate image dimensions before processing.

### 9. EXIF Metadata Not Stripped (Vuln #183) -- MEDIUM
**Issue:** Uploaded images retain GPS coordinates and device information. Privacy risk for users.
**Action:** Strip EXIF data during upload processing using Pillow's `image.save()` without EXIF, or use `piexif.remove()`.

### 10. Password Change Does Not Invalidate Sessions (Vuln #189) -- MEDIUM
**Issue:** Changing password updates the hash but does not blacklist existing JWT tokens. Active access tokens (15-min lifetime) remain valid.
**Action:** After password change, blacklist all outstanding refresh tokens for the user via `OutstandingToken.objects.filter(user=user)` and blacklist them.

### 11. No REST API Field Filtering (Vuln #147) -- LOW
**Issue:** No `?fields=` query parameter support. Low severity as serializers already limit exposed fields.
**Action:** Consider adding `django-rest-framework-fields` for bandwidth optimization, especially for mobile clients.
